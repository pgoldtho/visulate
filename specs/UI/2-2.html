Developer Notes

This screen should allow the user to maintain tenant details that are not available on the agreement screen. It also acts as a history screen that a property manager can use to record details of telephone conversations they have with the tenant. The screen should also provide a read only summary of payments due and payments made by the tenant. (I'm not sure why these are showing in a smaller font in the UI mock-up)

Payments due are recorded in the RNT_ACCOUNTS_RECEIVABLE table, Payments made are recorded in RNT_PAYMENT_ALLOCATIONS.

The Actions button should take the user to the Actions screen where property managers can record details of any eviction proceedings they are taking against the tenant.
