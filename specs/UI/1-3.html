Developer Notes

The Expense and Maintenance screen is based on RNT_PROPERTY_EXPENSES and RNT_ACCOUNTS_PAYABLE. It should be available to managers and owners. Managers use the screen to record details of problems that require repairs or routine maintenance. Owners may record details of property taxes and insurance payments that they need to make.

The screen should have a navigator table at the top of the page. This should show all the RNT_PROPERTY_EXPENSES rows for a given year. Users select a year using the drop down list and a row using the radio button. This populates the details area of the screen. Users can edit or add new expenses in the details area.

Expenses may be recurring or one time items. A one time expense corresponds to a single event like a blocked toilet or roof repair. Each event may result in one or more invoices. These are recorded in the RNT_ACCOUNTS_PAYABLE table using the multi row block at the bottom of the screen. A list of values button should allow the user to select from existing RNT_SUPPLIER records. It should also provide the option to create a new supplier.

Recurring expenses created for scheduled events like having a contractor cut the grass. A check box is used to mark expenses are recurring and a drop down list identifies the recurring period (See screen for values). The recurring end date records the date that recurring should stop.

The RNT_ACCOUNTS_PAYABLE table is used to record details of invoices or other payments that need to be made. Each payable item must be classified by payment type. These are recorded in RNT_PAYMENT_TYPES. The payment type determines the depreciation schedule that should be applied when reporting expenses. Some items last longer than others. For example, a new roof or air conditioning unit will typically last 10 years. The associated costs should be spread out (depreciated) over a 10 year period.

Some expenses may be recoverable from the tenant. For example, the landlord may pay to have a broken window fixed and then charge the cost to the tenant. The recoverable flag should be used for this.
